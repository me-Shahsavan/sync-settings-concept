The insight that drove v3

In v1, the Push / Pull buttons worked — but users hesitated to click them. They didn't know what a sync would change or whether it could be undone, so a working feature went unused. That's not a reliability problem; it's a trust problem. v3 answers the question the user was actually asking — "what happens if I press this?" — with preview, a test-one option, and visible history.

Integration concepts this prototype demonstrates

Source of truthPer-object, not binary — e.g. which side owns stock for a matched location.
Directional syncSend-to vs bring-in are separate, explicit decisions — not one toggle.
FreshnessReal-time on change vs an hourly sweep — stated to the user, not hidden.
Change previewShow the blast radius ("12 updates") before committing a sync.
Conflict resolutionMissing customer / pricing / taxing scheme — each with a clear next action.
Recovery policy"For future conflicts, do this without asking me" — create, adjust, or stop & error.
Safe vs human-reviewSome mismatches auto-resolve; some require a person. The UI encodes that.
VisibilityLast sync, recent imports, and per-item status build operator trust.
Self-contained HTML/CSS — no build step. Open either file directly in a browser.
